# {{site.data.keyword.cloud_notm}} の開発手法

{{site.data.keyword.cloud}} では、{{site.data.keyword.watson}} サービス・アプリケーションを開発して {{site.data.keyword.cloud_notm}} で実行するための手法がいくつかあります。どの手法でも機能や動作は同じなので、一番やりやすい方法を選択してください。

{{site.data.keyword.cloud_notm}} の外部でアプリケーションを開発する場合は、ここに挙げる方法を使用する必要はありません。ただし、{{site.data.keyword.watson}} サービスのインスタンスを作成するために {{site.data.keyword.cloud_notm}} のアカウントが必要です。
{: tip}
{: shortdesc}

## {{site.data.keyword.cloud_notm}} Web インターフェース

{{site.data.keyword.cloud_notm}} サイトでスターターを使用すれば、アプリケーションを短時間で作成できます。*スターター*とは、事前定義のサービスとアプリケーション・コードが含まれているテンプレートであり、特定のビルドパックで構成されています。スターターには、ボイラープレートとランタイムという 2 つのタイプがあります。詳細については、[Cloud Foundry アプリケーションの作成](/docs/cfapps/index.html)を参照してください。

## Eclipse IDE

Eclipse 統合開発環境 (IDE) を使用して、{{site.data.keyword.cloud_notm}} アプリケーションの作業を実行できます。詳細については、[{{site.data.keyword.cloud_notm}} 用の IBM Eclipse ツールを使用したアプリケーションのデプロイ](/docs/manageapps/eclipsetools/eclipsetools.html)を参照してください。

## Cloud Foundry CLI

Cloud Foundry `cf` コマンド・ライン・インターフェース (CLI) ツールを使用して、ローカル・システムで {{site.data.keyword.cloud_notm}} のサービスとアプリケーションの作業を実行することができます。詳細については、[Cloud Foundry コマンド・ライン・インターフェース](/docs/services/watson/getting-started-cf.html)を参照してください。
